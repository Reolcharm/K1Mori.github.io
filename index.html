3. chapter05 广告投放系统的开发

3.1 广告投放系统的核心功能

投放系统是比较简单的模块，其核心实现的功能就是对广告数据（各个表）进行增删改查，即能够让用户（广告主/代理商）对数据进行查看、上传、修改与删除

3.2 业务功能实现的技巧与思路

思路 1: 参数状态通过枚举获得

@Getter
public enum CommonStatus {
 /**
  * 用户, 计划, 单元等等的有效状态
  */
 VALID(1, "有效状态"),
 INVALID(0, "无效状态");
 private Integer status;
 private String desc;

 CommonStatus(Integer status, String desc) {
     this.status = status;
     this.desc = desc;
 }
}

在 po 中通过

this.planStatus = CommonStatus.VALID.getStatus();

来获取状态 1.

思路 2: 请求和响应封装成对象的思想

  1. 将请求和响应需要的参数封装成一个 vo, 然后在service 层填充该 vo, 再被通用 common 包下的 public class CommonResponseDataAdvice implements ResponseBodyAdvice拦截, 又一次填充后, 统一响应给前台.

  2. 在封装中对各个参数进行校验, service 层直接调取校验结果进行判断 service 层这样进行判断.

思考:

更重要的是接口方法, 常量等的设计, 然后是对具体业务的梳理

1. Redis 的 数据存储类型

  • String

    • 可以包含任何数据,e.g. jpg 图片或者序列化的对象

    • 应用: web 应用中记录 每天用户访问站点的次数

  • Hash

    • 特别适合存储 对象
  • List
    • 栈, 最新消息排行榜, 越新的消息越能排前显示
  • Set
    • 无序, 唯一(去重)
    • 应用: 微博中, 将所有关注人和所有粉丝存到一个集合中, redis 为集合提供了求交集, 并集等操作, 非常容易实现共同关注, 共同喜好等功能
  • Sorted Set
    • 全班成绩(排行榜);
    • 带权重的队列, 普通消息的score 为 2, 重要消息为 1, 按score的大小获取工作任务,让重要任务优先执行

How to write blogs?

Under my blog’s root director：

  1. hexo new post "article title" –> New a .md work;
  2. \source\_posts –> Find the source work, open and recording what you want;
  3. Open the git Bash under my blog’s root director
    1. hexo g生成
    2. hexo s预览
    3. hexo d发布
